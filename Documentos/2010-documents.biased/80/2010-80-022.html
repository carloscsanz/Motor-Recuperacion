Load the file into a bitmap, use Bitmap.SetResolution and save the file
again.

Remember that images are kept open so if you want to save back to the same
filename you'll need to open it from a stream and explicitly close the
stream again. See the GDI+ FAQ for more details.

In addition, opening and re-saving JPEG files cause compounded degradation
of image quality due to the lossy compression algorithm used. You may wish
to save the file at 100% quality so as not to degrade it too much but this
will create a corresl=pondingly larger file. See the GDI+ FAQ for details on
how to save a JPEG with a specific compression ratio.

Bob Powell [MVP]

When an image is initially loaded from a file such as
Image.FromFile(<filename>) the file is opened and the image size etc made
available but the image bits may not be read until the picture is displayed.
This keeps the file open and it remains so for the life of the object.
Attempting to save a modified image back to the same filename throws an
exception.

To overcome this problem you can open the file using a stream, use
Image.FromStream to load the file and explicitly close the stream to release
the file lock.
