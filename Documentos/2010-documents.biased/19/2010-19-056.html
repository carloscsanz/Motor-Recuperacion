There is a program out there called FRAPS (www.fraps.com) that runs in the background (1-2% performance hit) that will tell you the FPS of most Direct3D games. Thought it was neat and tried it. It works pretty good. This lets you test the games you might play and see how fast your system is. 30 FPS is OK, 45 is good and 60+ is great. Try the games you play and see what you get.

This is a real world benchmark and not a synthetic benchmark.

Quake3 does stress the CPU also, but you must turn down the resolution because most graphics cards will hit there fill rate first. Try 640x480 resolution. Mess with things like dynamic lighting, marks on walls, high geomitry... These hit the CPU pretty hard. I should know, my K6-2 450 gets hit pretty hard, but doesn't drop below 35FPS often and maxxes at about 52.

One thing you dont do is base stuff on seti. It changes with slight adjustments to the memory or cpu. It is a chipset, memory benchmark. The faster the cpu the better times you get. But i got 6.9 hours with a simple memory change.


Sandra I like as it doesnt change per configuration. I bench and bench and benched some more and with any change in config it never changed the score. So either use something that uses a cpu and not a whole system, if you use a program that changes per config then dont bother benchmarking. CPU only benchmarks are better... Seti isnt a bench but it isnt cpu only if you use it to bench.
