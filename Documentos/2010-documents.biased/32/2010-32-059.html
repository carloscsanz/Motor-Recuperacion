Web Style Guide : XHTML 1.0 Tags

Tags Supported in XHTML 1.0 Strict

Here are some HTML 4.01 tags defined in XHTML 1.0 Strict DOCTYPE declaration. Defined tags will validate in W3C's XHTML validation engine. XHTML 1.0 Strict requires the document to be well-formed, and is the same as HTML 4.01 Strict in XML form. Strict specification imposes us to use CSS formatting instead of obsolete HTML styles.

the header tags
  • <h1> & </h1> reserved for the site/default title.
    (e.g. printer-friendly version page’s site title “De La Salle University, Manila”)
  • <h2> & </h2> à reserved for the section title.
  • (e.g. section title such as Academics, Colleges, College of Computer Studies).
  • <h3> & </h3> can be used for the content title (previously known as “pageTitle”).
  • <h4> & </h4> can be used for the content sub-titles.
  • <h5> & </h5> can be used for the series of indented content sub-titles.
  • <h6> & </h6> can be used for the series of non-indented content sub-titles.
the paragraph tags
  • <p> & </p> can be used for the content paragraph or the “body texts”.
  • <dl> & </dl> for the definition list tag. the parent tag or delimiter of <dt> & <dd> tags.
  • <dt> & </dt> can be used for the definition term tag. the child tag of <dl> tag.
  • <dd> & </dd> can be used for the definition description tag. the child tag of <dt> tag.
 the list & blockquote tags
  • <ol> & </ol> can be used for ordered or numbered list.
  • <ul> & </ul> can be used for unordered or bulleted list.
  • <li> & </li> can be used for list item as child tag for ordered & unordered lists.
  • <blockquote> & </blockquote> can be used to create block indented style.
the table tags
  • <table> & </table> can be used for tabular data.
    • <table summary=“title”> can be used as table attribute with a value to enhance table data accessibility.
  • <caption> & </caption> can be used for to provide a caption as title header for every tabular data.
  • <tbody> & </tbody> can be used to enclose the actual table data elements (the tr & td) as table body within a parent table tag (<table>).
  • <thead> can be used to enclose the specific table data elements (the tr & td) as table header within a parent table tag. this cannot be used inside the tbody elements.
  • <tr> can be used to create table row.
  • <td> can be used to create table data.
the image tag
  • <img src="image1.png" class=“selector”>
    Note: Use pre-defined class selectors to control image alignment, borders & paddings.
the form tags
  • <form> & </form> can be used for the form field delimiter.
  • <input> & <select> for the form's text field and drop-down menu.
  • <input id="textfield"> can be used this for a specific form’s text field.
the <div> & <span> tags
  • <div> & </div> can be used for space division layouting (structural element) instead of tables.
  • <span> & </span> can be used for a specific inline space division (structural element).
the <abbr> & </acronym>
  • <abbr> & </abbr> can be used to indicate an abbreviated form (e.g. DLSU, Mnla, Phil)
    Note: This tag is not supported in MSIE browsers.
  • <acronym> & </acronym> can be used to indicate an acronym (e.g. ITC, CCS, CLA)
the <code>, <tt>, <samp>, <var> & <kbd>
  • <code> & </code> can be used to demonstrate code examples within XHTML pages.
  • <tt> & </tt> can be used to format text into teletype or mono-space.
  • <samp> & </samp> can be used to show sample output from programs and scripts.
  • <var> & </var> can be used to designate a program parameter or variable.
  • <kbd> & </kbd> can be used to signify text to be entered by the user using the keyboard.
the <del>
  • < del> & </ del> can be used to demonstrate a deleted single word or phrases. Use this as alternative element to cross out ( strikethrough effect) a word or phrases.
the <ins>
  • <ins> & </ins> can be used to demonstrate inserted word or phrases. Use this as alternative element to demonstrate an underlined word or phrases.
the <b> & <strong>
  • <b> & </b> can be used to mark up text in bold.
  • <strong> & </strong> can be used to mark up text in strong.
the <big> & <small>
  • <big> & </big> can be used to mark up text bigger in size.
  • <small> & </small> can be used to mark up text smaller in size.
the <cite>, <i>, <em> & <dfn>
  • <cite> & </cite> can be used to mark up text as a citation such as titles of publications or references.
  • <i> & </i> can be used to mark up text in italic.
  • <em> & </em> can be used to mark up text with emphasis.
  • <dfn> & </dfn> can be used to mark up text with definition.

Tags Not Supported in XHTML 1.0 Strict

Here are some HTML 4.01 deprecated tags & attributes that are undefined in XHTML 1.0 Strict DOCTYPE declaration. "Undefined" means not supported/allowed . Undefined tags may cause error in W3C's XHTML validation engine.

the <font> + 'color', 'size' & 'face' attributes
  • <font> & <font color=“#996633” size=“3” face=“Arial, Helvetica, sans-serif”>Lorem ipsum dolor sit amet</font>
the 'border', 'align' & 'name' attributes
  • <img src=“image1.png” border=“0” align=“left” name=“image1” />
    Note: Image's 'border', 'align' & 'name' attributes are not allowed in XHTML 1.0 Strict document. Use css custom selectors for image instead.
the <s> tag
  • <s> & </s> is use to cross out (strikethrough) a single word or phrases.
    Note: This tag is not allowed in all XHTML document. You must use the <del>delete</del> tag to get the strikethrough effect for texts. This is usually fixed by using CSS text-decoration: line-through to achieve the desired effect instead.
the <u> tag
  • <u> & </u> is use to demonstrate underlined word or phrases.
    Note: This tag is not allowed in all XHTML document. You must use the <ins>inserted</ins> tag to get the underlined effect for texts. This is usually fixed by using CSS text-decoration: underline to achieve the desired effect instead.
the 'target' attribute
  • <a href=“#” target=“_parent”>Lorem ipsum dolor sit amet</a>
    Note: You must use the “XHTML 1.0 Transitional” document type to get the “target” attribute, or you can use 'onclick' event to mimic the behavior of controlling a link to open a new browser window or whatever you wanted to without using 'target'. For example:
    • <a href="page1.asp" onclick="return !window.open(this.href);"> or <a href="page1.asp" onclick="window.open(this.href); return false">
the 'marginheight' & 'marginwidth' attribute
  • <body marginheight=“0” marginwidth=“0”>content</body>
    Note: These attributes are vendor proprietary extensions & not allowed in all XHTML documents. This is usually fixed by using CSS to achieve the desired effect instead.
the 'frameset' tag
  • <frame> & <frameset> are also not allowed in XHTML 1.0 “Strict” document type with a document that uses frames. You must use the “XHTML 1.0 Frameset” document type to get the “frameset” tags.

Notes:

  1. In XHTML document, don't forget to close single tags (e.g. img, br) with “/>” instead of “>” only, or perhaps you're meant to “self-close” a tag by enabling XHTML compliant document.
    For example:

    <img src="image1.png" class="cb-image-left" />
    <br />
     
  2. Some error may also result if the element itself is not supported in the document type you are using, as an undefined element will have no supported attributes; in this case, see the element-undefined error message (in W3C's validation result) for further information.
     
  3. Check the spelling and case of the element and attribute. Remember XHTML is all lower-case. (e.g. use onclick instead of onCLick)
     
  4. Use CSS instead as much as possible.
     
  5. Use the proper <div> tags assignment with the <form> & form's child elements.
    For example:
     
    <div id="idname">
     <form class="search" method="get" action="/cgi-bin/search/">
      <div class="classname">
       <input type="text" name="textfield">
       <input type="submit" name="submit" value="Submit">
      </div>
     </form>
    </div>
