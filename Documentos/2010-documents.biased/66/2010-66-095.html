    issue 1808 (entry.Update() pushes the update then sets all updated field...) reported   -   Name of API affected: Calendar Data API , using the C# library (did not try others) Issue summary: Calling Update on an EventEntry twice results in an exception. The reason seems to be that the new Etag is not read back from the server, even though other fields that change with the first call to Update() do have their new values. (The calender used is not the main calender. This might be important, at least if 'non-main calenders' and 'folders in google docs' are somewhat alike, see http://groups.google.com/group/google-documents-list-api/browse_thread/thread/659935273439d818/8a040eff0ce90ee7?hl=en&ie=UTF-8&q=Mismatch:+etags+google#8a040eff0ce90ee7 ) Steps to reproduce issue: Run this sample code in a test app: CalendarService service = new CalendarService("CalendarTESTApp"); service.ProtocolMajor = 2; service.ProtocolMinor = 1; then get a feed with an entry, and then (with textBox1 some TextBox to report to): if ( calenderFeed.Entries.Count > 0 ) { EventEntry entry = (EventEntry) calFeed.Entries[0]; textBox1.AppendText( "--------------------------" + Environment.NewLine); textBox1.AppendText( "Edited:" + (entry.Edited == null ? "null" : entry.Edited.Value) + Environment.NewLine ); textBox1.AppendText( "Etag: " + (entry.Etag == null ? "null" : entry.Etag.ToString()) + Environment.NewLine ); textBox1.AppendText( "Updated:" + entry.Updated.ToString() + Environment.NewLine ); entry.Title.Text = entry.Title.Text + entry.Sequence.IntegerValue.ToString(); entry.Update(); textBox1.AppendText( "UPDATED ENTRY" + Environment.NewLine); textBox1.AppendText( "Edited:" + (entry.Edited == null ? "null" : entry.Edited.Value) + Environment.NewLine ); textBox1.AppendText( "Etag: " + (entry.Etag == null ? "null" : entry.Etag.ToString()) + Environment.NewLine ); textBox1.AppendText( "Updated:" + entry.Updated.ToString() + Environment.NewLine ); //make sure we have the most recent version of the entry EventEntry theRereadEntry = (EventEntry) service.Get( entry.SelfUri.ToString() ); textBox1.AppendText( "RE-READ UPDATED ENTRY" + Environment.NewLine ); textBox1.AppendText( "Edited:" + (theRereadEntry.Edited == null ? "null" : theRereadEntry.Edited.Value) + Environment.NewLine ); textBox1.AppendText( "Etag: " + (theRereadEntry.Etag == null ? "null" : theRereadEntry.Etag.ToString()) + Environment.NewLine ); textBox1.AppendText( "Updated:" + theRereadEntry.Updated.ToString() + Environment.NewLine ); entry.Update(); //This will throw due to Etag mismatch } sample output: Edited:2010-01-24T21:11:26.000Z Etag: "EE0PRwJJeCp7IGA6WhJT" Updated:24/01/2010 22:11:26 UPDATED ENTRY Edited:2010-01-24T21:31:38.000Z Etag: "EE0PRwJJeCp7IGA6WhJT" Updated:24/01/2010 22:31:38 RE-READ UPDATED ENTRY Edited:2010-01-24T21:31:38.000Z Etag: "EE0PRgxIdip7IGA6WhJT" Updated:24/01/2010 22:31:38 Expected output: the Etag after 'UPDATED ENTRY' should be different from the one before it, so that the next call to entry.Update() succeeds. Actual results: Instead, the Etag is the same. Note how the 'Edited' and 'Updated' properties do change immediately with the Update() call! The re-read entry shows that a new tag does exist. The next call to entry.Update() then throws an error about the Etag. E.g. "Mismatch: etags = [&quot;EE0PRwJJeCp7IGA6WhJT&quot;], version = [63400051898]" Note that the re-reading indicates a workaround, but if the new Edited and Updated values are fetched from the server, why not the Etag as well?
