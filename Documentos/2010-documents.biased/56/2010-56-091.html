As long as the game has AA and AF options, its best to leave everything else in CCC on application settings.

Make sure that box is selected under AA (This matters, even if you have it using the application's AA and make sure Multi-Sample anti aliasing is selected with the slider. Super Sampling and Adaptive can be problematic in some games.

Also, ensure that Catalyst AI is checked and crossfire is enabled.

Lastly, can we see a screenshot of the GPU-Z sensors tab?

I don't think your processor is to blame at all.

for the gpu score of 25k, they generally do a P test and OC the hell out of it. on my setup i score about 23-24k with both my cards at 960/1125, so your score is fairly good at those clocks.

i7 is only a small bottleneck tbh, its not much of a bottleneck if you play at resolutions higher than 1680x1050. my cpu is more of a bottleneck, my phenom ii x4 at 3.8ghz only scores 13k cpu score in the p test...

extreme scores are better for comparisons with a card of your level as the test runs at a higher resolution which minimises cpu bottlenecks.

there should be a few threads about dragon age that you can find here that should provide some useful tips. i dont think bc2 has a crossfire profile just yet, as the game isnt out in some countries yet so id wait for the next driver/crossfire profile batch from ati for that. and as for gta4... it generally runs like crap on a lot of computers. mine averages 55fps, with frequent dips to 30fps, but i have a terrible cpu (for gta4 anyway). if you google optimising gta4, you can find a whole bunch of mods that improve the fps in the game (at the cost of loading times in some cases). are you sure you arent getting some hdd lag in gta4? what hdd are you running?

personally id say run it at stock clocks and volts and see if it crashes. sometimes OCs are only stable for certain games and certain programs. ie. i can run my cards at 960/1125 with no overvolting and its perfectly stable for mass effect 2, but it needs a few more mV to be stable in mw2.

another thing to check is that the card is seated in the pcie slot properly. if it isnt then it can cause random crashing (if youre lucky enough to get it to boot with it not seated properly that is)
