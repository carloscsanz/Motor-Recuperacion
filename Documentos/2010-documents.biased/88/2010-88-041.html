Simple HTML Tutorial - Part 3 (Basic Code & Markup Differences)

by jbarnett on May.02, 2009, under Tutorials, Web Design

So probably having found me through Google, you probably saw (2) similar but different types of code in your search for HTML Tutorials. HTML and XHTML. Part of what makes learning how to design websites with code late in the game, is that standards have changed, and with change comes new standards. HTML is the old web standard, while XHTML is the new web standard. They are both similar, but of course different.

Also just so you know, HTML stands for “Hyper Text Markup Language”, while XHTML stands for “eXtensible Hyper Text Markup Language”. You don’t have to remember those terms, but it’s always helped me push myself to continue learning code when I’m reminded that acronyms usually come from logical terms.

So whats the difference between HTML and XHTML exactly?

HTML consists of old fashioned “tags”, such as:

<table></table>
<tr></tr>
<td></td>
<font></font>

While XHTML consists of present recommended “tags”, such as:

<div></div>
<span></span>
<ul></ul>
<li></li>
<ol></ol>

What do I recommend?

I recommend that you focus on XHTML if you can follow it, but also know your basic HTML for the sake of being able to get your hands dirty with sites coded in HTML. HTML is the code that a lot of designers fall back on (even though they shouldn’t) when XHTML isn’t working for them. A number of “WYSIWYG” editors still code in HTML so when using CMS (Content Management Systems) sites, its good to know how to fix code generated by “visual editors”.

What will I be teaching in my tutorials?

I’ll be covering basic HTML, getting your site online, tracking statistics, and then finally move onto XHTML. Knowing your basic HTML will help you learn XHTML and understand the concept behind it.

So lets proceed in learning HTML by introducing you to HTML tables, columns, and rows. In theory its easy to understand, and with practice, easy to code. Let’s use an object such as a Picture Frame as an example.

Tables, Columns, and Cells in HTML

With HTML I like to think of the website as a blank grid. Everything is “square” based in HTML. So for example, lets begin by talking about the <table> tag. In relation to our “Picture Frame” that we’re building with HTML, the <table> tag would be the frame.

So lets make the frame of our picture frame.

<body>

   <table>
      Some text inside our picture frame.
   </table>

</body>

Now if you save the file and refresh the page (or open the file in your browser if you don’t already have it open), you’ll notice that there’s no frame at all. In HTML, you have to be specific. So if we want to have a frame, we would do so by “adding a border” in HTML. Now theres an “HTML” approach to this, and a “CSS” approach to this. So I’m going to skip over the HTML approach and show you the more “proper” “CSS” approach. By the way, CSS stands for “Cascading Style Sheets”.

Go ahead and create a new CSS file and save it in the same folder as your HTML file. If your using Notepad or TextEdit, make sure to save the file as such: “styles.css” (without quotes). Once you’ve created and saved your CSS file (note that its currently blank), we need to tell our HTML file to use it.

Add this line right before the opening “body” section:

<link rel="stylesheet" type="text/css" href="/styles.css" />

so it would look like this:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<title>My Website!</title>
</head>

<link rel="stylesheet" type="text/css" href="/styles.css" />

<body>

   <table>
      Some text inside our picture frame.
   </table>

</body>
</html>

Ok so now lets assign our table an “id” so that we can add a border via CSS without adding borders to every table on our site.

   <table id="frame">
      Some text inside our picture frame.
   </table>

So here’s our CSS file -note that anything inbetween this: /* this is a comment and is not rendered */

/* styles.css - remember this is a comment and the browser skips over this (does not render) */
#frame {
   border-width: 10px;
   border-style: ridge;
   border-color: #00cccc;  /* HTML colorcode for a blue */
}

If you save the file, we should have something that looks like this:

Some text inside our picture frame.

So now we should have an HTML file that as a table with the id of “frame”, and a CSS file that is giving any “element” that has an id of “frame” a 10 pixel blue border.

Simple HTML Tutorial - Part 2 (Setup)

by jbarnett on Apr.20, 2009, under Tutorials, Web Design

So lets get you setup to write some code.

I usually recommend a program that will color code your HTML markup, but if you’d rather start without color coded markup, then you can just open “Notepad” (for Windows users), or “TextEdit” (for Mac users).

MAC USERS: If your using TextEdit, select “Plain Text mode”. Its listed in one of the dropdowns in the toolbar when you have TextEdit open. If you can’t find it, you can try this: TextEdit Plain Text Help

A great free HTML program (that I used when I was new to HTML) was Chami HTML-Kit. Its a great alternative to Adobe Dreamweaver if you can’t afford or don’t have Dreamweaver.

Assuming your using NotePad or TextEdit, copy the entire code below into your new file
(use the icon).

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<title>My Website!</title>
</head>

<body>
This text is visible to the world. Hello world!
</body>
</html>

Now lets preview it in our browser. Go to FILE > Save As.

In Windows: Navigate to where you want to save it (such as the Desktop). Type ‘index.html’ without quotes into the “File name” field. Change “Save as type” dropdown to be “All”. If there is a dropdown for “Encoding”, choose “UTF-8″, otherwise, just click “Save”.

In Mac OSX: Navigate to where you would like to save it. Type ‘index.html’ for the “File name” and click “Save”.

Now find the file that you saved and open it. If it opens in Notepad or TextEdit (or if it simply brings you to your already opened file), then you’ve saved it as a .txt file rather than a .html file. It’s important to save the file as a .html file so that its recognized by the browser.

The file should open in your browser with the following text displayed:

This text is visible to the world. Hello world!

If it didn’t open in your browser, you’ll need to make sure its a .html file and not a .txt file. For Windows users, you can navigate to START > Control Panel > Folder Options (classic view), and under the “View” tab, uncheck “Hide extensions for known file types” and click OK.

If I find a lot of people are having problems getting it to work, I’ll do what I can to implement screenshots. Just leave a comment to let me know.

So you have it opened in your browser. Lets modify the text accordingly to say something else.

If you closed your NotePad or TextEdit, you can reopen it by either renaming the file to have the extension .txt (instead of .html), or right clicking on the file > Open With… > Notepad (or TextEdit).

Once you have it open, find the text: 

<body>
This text is visible to the world. Hello world!
</body>
</html>

You’ll notice that its located in between <body></body> tags. Everything in between those tags is where you’ll primarily put most of your code (as its the visual portion of the site in simple terms).

Let’s go ahead and make the text bold:

<b>This text is visible to the world. Hello world!</b>

As you can see, the only thing we have to do to make the text bold, is to add a <b> to the beginning of the text, and a <b> to the end. This tells the browser that only the text inbetween those 2 <b> tags are to be bolded. Notice the forward slash to indicate where bold text should “End” while not having the forward slash would indicate “Beginning” bold text.

Go ahead and save the file, and either refresh the window you have open (if you have the site already opened in the browser), or re-open the file after having saved it.

The result should come out something like this:
This text is visible to the world. Hello world!

So maybe we don’t want all the text bolded, so lets make it so that only “Hello world!” is bold.

This text is visible to the world. <b>Hello world!</b>

You can also italicize or underline the text using <i> instead of <b>
You can put the text below right after the “Hello World” if you like. Give it a try!

<i>This text is italicize.</i>
<u>This text is underlined</u>

Finally, lets introduce you to line-breaks. Let’s TRY to make “Hello world!” go to the next line:

This text is visible to the world.
<b>Hello world!</b>

Now even though we’ve put it on its own line in the code, doesn’t mean the browser will recognize it. If you save your file and refresh your page, you’ll notice that the text hasn’t changed. In HTML, you’ll have to remember that (1) only one space is recognized (therefore spacebar twice in a row is recognized as only one when viewed in a browser); and (2) text only goes to the next line if its too long or if you tell it to do so using code.

So lets add the <br> tag to our list:

This text is visible to the world.<br />
<b>Hello world!</b>

If you save and refresh the browser, you should notice that “Hello world!” has moved to the next line. Good job!

Just in case you wanted to know, the <br /> is XHTML and <br> is HTML. Both do exactly the same thing; however, in XHTML (current web standard), ALL “open” tags must have a close tag. Therefore <br /> is both an “open” and “close” tag in itself while <br> IS NOT.

I also know that there’s a lot of “information” that’s coming at you. I’m doing my best to try and keep it minimal. In general I’ll do my best to only cover what I believe you should know.

Apply the Knowledge!

1) First, make the first text line bold.
2) Now italicize the text without taking away the bold.
3) Add a line-break so the second line is on its own line.
4) Finally, make the text “web design” underlined.

I love web design!
This belongs on its own line!

When your all done, your HTML markup should look like this:

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
<title>My Website!</title>
</head>

<body>
<i><b>I love <u>web design</u>!</b></i><br />
This belongs on its own line!
</body>

</html>

Note that the <br /> can actually be on the first line or the second line.

Next, I’ll describe the difference between HTML and XHTML, and will be diving into HTML “tables”, “columns”, and “rows”.

Simple HTML Tutorial - Part 1 (Introduction)

by jbarnett on Apr.19, 2009, under General Web Design, Tutorials, Web Design

I believe at some point, knowing basic HTML will be a basic job requirement for a larger number of jobs in the workplace. Similar to how Microsoft Word or Excel skills are required for some office jobs, managing web content may be one of your everyday job function. It also adds job security if your the only one that has HTML knowledge.

There are a lot of tutorials out on the internet that can be of a lot of help. But the thing about tutorials is that there only as good as the person writing them. I find that many tutorials go in the wrong direction or “assume” things. So I’m going to give the public, my own set of help articles that will get you started designing web sites.

Who are these articles intended for? I would recommend my basic (x)HTML tutorial to those whom have no knowledge of HTML, or only knows the extreme basics (such as styling text). As we move forward, I’ll cover more advanced topics. I’m also going to include very practical tutorials such as purchasing a domain name, uploading files, tracking site statistics using Google, installing applications like Wordpress or ExpressionEngine, and so forth.

In addition, I’m going to do my best to try not to waste your time teaching you things that are of old standards, rather, I’m going to work on teaching you how to code based on existing web standards. By web standards I’m talking about new coding techniques that apply to the present, and not the past. Why is it good to write code (also called markup) using proper web standards? When you write markup using present standards, your sites will display properly across multiple browsers and operating systems, it will become easier to add or remove markup, and not to mention, its more likely to be indexed (stored) in Google and displayed in search results. After all, a website is meant to be looked at.

So with that said, enough talk, let’s dive in.
