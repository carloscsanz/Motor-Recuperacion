Hi Seangr,

> This is the sample code, The Connect does not return, It seems it is in a
> different Thread. I tried with RAPI.Connect(True, 60), still did not
> returned
> any value.

The problem is your use of Connect. The way you call it will cause it to
block until a connection is successfully made. The timeout argument is
ignored if you pass True for the first parameter.

I would try to pass False as the first parameter (make the call to Connect
async and return immediatly). You could then use the RAPIConnected and
RAPIAsynchronousTimeout events to provide your desired prompt logic.

The following C# should demonstrate what I meant. I leave it as an exercise
to convert to VB.NET

RAPI r = new RAPI();

public Form1()
{
InitializeComponent();

// Hook up the various event handlers
r.RAPIConnected += r_RAPIConnected;
r.RAPIAsyncronousTimeout += r_RAPIAsynchronousTimeout;
}

private void button1_Click(object sender, EventArgs e)
{
// Start a connection attempt with a 5 second timeout
r.Connect(false, 5);
}

void r_RAPIAsynchronousTimeout()
{
// This is called if the connection attempt times
// out so prompt the user and try connecting again
MessageBox.Show("Please connect your PDA via ActiveSync");
r.Connect(false, 5);
}

void r_RAPIConnected()
{
// This is called once a device is connected via
// RAPI. You can place your code which uses
// RAPI to interact with the device here.
MessageBox.Show("Your device is connected");
}

Hope this helps,
Christopher Fairbairn
