    You could use a SSH (secure shell) client wrapper for .NET to connect to the remote UNIX machine and execute commands to run your script.

     

    Have a look at: http://www.codeproject.com/KB/IP/sharpssh.aspx

     

    If your UNIX machine doesn't have a SSH daemon running/installed then you could also look for a Telnet client wrapper for .NET.

     

    For Telnet check out: http://dotnettelnet.sourceforge.net/

     

    Google keywords for SSH: "SSH C#" and "Telnet C#". I like to search for C# versus .NET so I don't get back .net TLD hits.

     

    Good luck,

    Scott R. Jones

    here is the rfc for telnet.  That has all the information you need if you want to implement your own telnet client or server.  It might be very difficult for you so I recommend that you just use a 3rd party opensource/free component for this.  Ssh is super complex and would be extremly hard for you to implement.  Again 3rd party opensource components are the way to go.  There are probably many commerical libraries you can purchase as well.

     

    Telnet Rfc

    http://www.faqs.org/rfcs/rfc854.html

     

     

    Opensource Telnet Libraries

    http://dotnettelnet.sourceforge.net/

    http://www.thoughtproject.com/Libraries/Telnet/

     

     

    Opensource ssh lib

    http://www.tamirgal.com/home/dev.aspx?Item=SharpSsh
