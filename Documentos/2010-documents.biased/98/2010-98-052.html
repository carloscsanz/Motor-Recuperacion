Body tag attributes
HTML has accumulated a large number of presentational attributes over the year. These have for the most part been deprecated in favour of style sheets. The BODY element can take several attributes that will style your page. Here is the list of them: <body background="tile.gif" bgcolor="#ffffff" text="#000000" link="#0000cc" vlink="#660000" alink="#cc0000">   </body>

  1. background - page's background image (will be tiled)
  2. bgcolor - page's background color
  3. text - assigns a colour to the text
  4. link - link color
  5. vlink - visited link color
  6. alink - active link color
You can download some free background images from http://www.backgroundsarchive.com/

How do you specify the colour you want?
The simplest is to use a standard colour name, but there are only 16 of them (which correspond to the basic VGA set on PCs): aqua, black, blue, fuchsia, gray, green, lime, maroon, navy, olive, purple, red, silver, teal, white, and yellow.

More specific colours can be set by using hexadecimal (0...9A...F) values, that represent colours as three groups of two hexadecimal numbers for the Red, Green and Blue elements (RGB).
Another way to specify the colour is using the decimal system of colour values, each of the red, green and blue components are specified as three groups of numbers, from 0 to 255, separated by a comma.
Here are some examples: 

Colour			Decimal			Hexadecimal
Black			0,0,0			#000000
White			255,255,255		#FFFFFF
Red 			255,0,0			#FF0000
Green			0,255,0			#00FF00
Blue			0,0,255			#0000FF
Aqua			0,255,255		#00FFFF
Yellow			255,255,0		#FFFF00
Fuchsia			255,0,255		#FF00FF
NS and IE implement a palette of 216 browser-safe colours, which will always display properly whatever the platform and whatever the colour depth on the screen. The palette is constructed from colours where red, green and blue are each restricted to the following hexadecimal values: 00, 33, 66, 99, CC, FF.

Creating Lists
One useful feature of HTML/XHTML is the ability to create lists. Lists are useful in most any circumstance where you want to itemize or categorize things. To create an Unordered (bulleted) List you need to add a <ul> and a </ul> tag at the beginning and the end of the list. Ordered (numbered) Lists have <ol> tags instead of <ul> tags. To separate single List Items use <li>...</li> tags. There are special settings that you can use to customize the lists on your page.

A single list item can be a single word, or it can be many paragraphs long, complete with supporting multimedia elements.

If you do not like the default bullet for the Unordered List, HTML provides the type attribute, which allows you to choose between whether you want the bullet to be a:
  • circle
  • disc
  • square
The type attribute has been replaced by style sheets, but it can still be used to be compatible with older browsers. The type attribute can either go in the list tag to change the bullet for the entire list or in the list item tag to change the bullet for a single item. The use of style sheets allows for even more options, such as using graphics instead of bullets, or using no bullets at all.

An Ordered List is a list where the items have a specific order or sequence that needs to be noted, for instance a sequence of steps that need to be followed to successfully complete a task, or a list of rankings in a contest. The numbers in an ordered list start at one (1) and proceed upwards for each list item unless the browser is told otherwise.

The <ol> tag is used to delimit a set of list items. It has various attributes you can set to make ordered lists more flexible. One (deprecated) attribute is start, which allows you to define a starting number other than one (1). This is useful if the list is broken up by other content, since it allows you to pick up where you left off.

Another related attribute is type, which works in more or less the same was as it does for unordered list. Here you give it a numbering system to count with. The valid values for type are: 

   Type Value		Style Used			Sample
	A		capital letters			A, B, C, D
	a		lowercase letters		a, b, c, d
	I		capital Roman numerals		I, II, III, IV
	i		lowercase Roman numerals	i, ii, iii, iv
	1		Arabic numerals (default)	1, 2, 3, 4

You can nest lists, something particularly useful if the list is an outline of some sort.

Nested unordered lists may rotate through the three bullet types as you move through nesting levels, depending on your browser.

Ordered lists, by default begin with Arabic numbers regardless of the level of nesting. However, you can use the type attribute or style sheets to override both of those and define the different levels to be whatever you want.

Note that properly nested lists means that everything within a list exists between an <li> tag and it corresponding end tag (</li>). This means that a nested list should be contained within a list item, not occur between list items. Most browsers will let you get away with nesting a list within a list without enclosing the nested list in a list item of the parent list, but it is not a well-formed coding statement.

Right: 
<ul>
  <li>Item 1</li>
  <li>Item 2
    <ul>
     <li>Sub-item 1</li>
     <li>Sub-item 2</li>
    </ul>
  </li>
  <li>Item 3</li>
</ul>
Wrong: 
<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <ul>
   <li>Sub-item 1</li>
   <li>Sub-item 2</li>
  </ul>
  <li>Item 3</li>
</ul>

Table Structure
Using tables, it is possible to display tabular data, create columns of text, precisely position various elements of your design, and solve many HTML design problems.

The TABLE element is a container, so it requires both the start tag and the end tag. Everything between is part of the table. The TR tag (also a container) designates the beginning of a row (horizontal) and each of the TD tags (also containers) marks the beginning of a column (vertical).

Let's start by defining the structure elements of an XHTML table.
  • table: this element makes up the entire table structure.
  • caption: this element is optional, but if it is included it must be the first element in the table.
  • thead, tfoot, and tbody: if these elements are used (they are also optional) they must be used in the order shown here.
  • tr: this element makes up the row structure of the table.
  • th and td: table header and table detail. They are always used in conjunction with the <tr& element. These two elements make up the cells of the table. TH element is like TD, only it bolds and centers the text
Grouping Tags
The grouping tags are new to HTML. They are recommended for use in XHTML, but are not fully supported in all browser. The grouping tags are:
  • <thead>
  • <tfoot>
  • <tbody>
The grouping tags allow you to organize and format tables by section. The <thead> specifies a header for the table. For a browser that recognizes this tag, it would allow the browser to do things like print up the header information at the top of each page if the table runs beyond one page. There can be only one <thead> grouping in a table and it should occur at the top of the table, before any other content.

The <tfoot> allows you to define a footer for a table in the same manner as a header. Once again there can be only one. The footer should appear in the code after the header and before the body. The problem is that older browsers do not know to move it to the bottom. Therefore, to be backward compatible, it should appear at the bottom of the table, after any other content.

The <tbody> tag is the last of this set and allows you to specify body sections in a table. There can be more than one body section in a table, allowing you to do sectional formatting. You can consider this tag to be the equivalent of a <div> tag for use in tables.

Customizing Tables The table itself becomes the size of all the elements it contains. If the elements within the table are not a fixed size, the table gravitates to the size of the window in which it is displayed, and gives equal space to each of the elements in the table.

The following attributes can be used with Table tags (Please note, that all these attributes will give us backward compatibility, they all can be set using CSS, that gives greater flexibility):

  • border - sets the thickness of the border surrounding the whole table (accepts integers, when set to 0 - there are no borders)
  • frame - used in conjunction with the border attribute, it lets us to decide where to put frame:
    • void - will draw no frame at all
    • above / below - will produce just a top / lower border
    • lhs / rhs - will give a left-hand / right-hand border only
    • hsides / vsides - will produce top & bottom / left & right borders
    • box / border - will give a full border
  • rules - allows us to decide how to draw dividing lines between various groups of cells:
    • none - will draw no borders around the cells
    • all - will draw a border around every cell
    • groups - will draw a border around tbody, thead, tfoot
    • rows - will draw a border between rows
    • cols - will draw a border between individual columns

  • bgcolor / background - to set a background colour / image (tiled), can be used with table, tr, td, th, thead, tfoot, tbody elements
  • width / height - to set the width /height of both the table and the individual cells. It can take a relative value (%), or an absolute value (px)
  • align - the default alignment is "left", but can be changed to "center" or "right"; can be used with table, caption, tr, td, th, thead, tfoot, tbody elements. When used within the <table> tag, the text will flow around the table (similar to alignment with the <img> tags). When used within caption, tr, td, th, thead, tfoot, tbody elements, controls the horizontal alignment of the text within the cell, can take an additional value of "justify"
  • valign - sets the vertical alignment of the text within the cell; also can be used within caption, tr, td, th, thead, tfoot, tbody elements. The possible values are: "middle" (default), "top", "bottom", "baseline". (with the <caption> valign can be set only to either "top" (default) or "bottom")
  • cellspacing - Determines the amount of space (an integer value, in px) to put between cells. This amount will be put inside the border.
  • cellpadding - Determines the amount of space (an integer value, in px) to put between the edge of a cell and its contents.

More about Tables will be covered in the Advanced HTML/XHTML course.
