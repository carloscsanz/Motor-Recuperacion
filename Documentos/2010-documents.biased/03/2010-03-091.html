Elements and Tags

Prior to discussing the essentials of XHTML it is important to gain an understanding of HTML structure. The following provides information concerning important concepts of HTML. Acquiring a fundamental comprehension of HTML structure will enhanced your inclusive understanding of XTHML and document structure.

HTML uses a series of markup tags that describe the structure and content of a Web page. When you view a Web page you are actually seeing your browser's interpretation or rendering of embedded HTML elements. Markup elements are comprised of a start tag and a possible end tag. Web authors use HTML tags to configure data into various formats such as paragraphs, tables, and lists. HTML specifies what each tag means and what the data between tags will look like. An HTML document therefore, is nothing more than a text file containing the information you want to publish and the markup instructions that tell the browser how to structure and present the information. Consider the following HTML element example:

<h1>HTML is easy...</h1>

Analyzing the above example reveals the following:

  1. Start tag:  <h1>
  2. Content or what we see onscreen:  HTML is easy...
  3. End tag:  </h1>

Tags are inclosed using angle brackets. To indicate an end tag insert a forward slash prior to the element type, e.g., </h1>.

Optional Tags

Some HTML element types according the HTML recommendation allow you to omit the end tag. Consider the following example for the paragraph, <p> tag.

HTML Example:  <p>The paragraph end tag is optional in HTML.

XHTML Example:  <p>The paragraph end tag is not optional in a XHTML document.</p>

Below is a list of common element types that do not require an end tag:

  • List Item:  <li>
  • Selectable Choice:  <option>
  • Paragraph:  <p>
  • Table Body:  <tbody>
  • Table Data Cell:  <td>
  • Table Footer:  <tfoot>
  • Table Header Cell:  <th>
  • Table Header:  <thead>
  • Table Row:  <tr>

Following the rules of XML; in an XHTML document all tags must be closed, hence, the end tag is added.

  • List Item:  <li>...</li>
  • Selectable Choice:  <option>...</option>
  • Paragraph:  <p>...</p>
  • Table Body:  <tbody>...</tbody>
  • Table Data Cell:  <td>...</td>
  • Table Footer:  <tfoot>...</tfoot>
  • Table Header Cell:  <th>...</th>
  • Table Header:  <thead>...</thead>
  • Table Row:  <tr>...</tr>

Empty Tags

Elements that do not have content are known as empty elements. For instance, the line break <br> tag is considered empty because its purpose is to terminate a line.

Below is a list of common element types where an end tag is forbidden:

  • Clint-side Image Map Area:  <area>
  • Base Font Size:  <basefont>
  • Line Break:  <br>
  • Sub Window:  <frame>
  • Horizontal Rule:  <hr>
  • Embedded Image:  <img>
  • Form Control:  <input>
  • Media-independent Link:  <link>
  • Metainformation:  <meta>
  • Named Property Value:  <param>

Following the rules of XML; in an XHTML document all tags must be closed. Therefore, to close an empty tag simply add a space and forward slash after the element type.

  • Clint-side Image Map Area:  <area />
  • Base Font Size:  <basefont />
  • Line Break:  <br />
  • Sub Window:  <frame />
  • Horizontal Rule:  <hr />
  • Embedded Image:  <img />
  • Form Control:  <input />
  • Media-independent Link:  <link />
  • Metainformation:  <meta />
  • Named Property Value:  <param />

Attributes

SGML provides that HTML elements may have associated properties know as attributes. Attributes consist of two parts; the attribute name and the attribute value or attribute/value pair (name=value).

General rules for attributes:

  • Attributes appear in the start tag prior to the > bracket.
  • Include as many legal attribute/value pairs separated by spaces and in any order.
  • Always enclose attribute values with double quotation marks.
  • Attribute values are not case sensitive.

Example:

<table border="1" id="table1" width="50%">

Below are commonly used attributes:

  • style: specifies style sheet information to a tag.
  • title: advisory text or Tooltip in many browsers.
  • class: associates a number of tags to a particular class.
  • id: document wide id, establishes a unique name for an element.

Comments

Comments can be inserted into your Web document using the following format:

<!-- comments -->

web page structure

Now that you have a basic knowledge of tags, let's review the structure of a Web page. The structure of a HTML document is quite simple. It includes the following basic structure in the order shown:

HTML Document Structure

<!DOCTYPE>
<html>
<head>
<title>...</title>
</head>
<body>
</body>
</html>

Below is a simple Web page example:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
<head>
<title>HTML Web Page</title>
</head>
<body>
<h1>This is an HTML 4.01 Web page</h1>
<hr>
<p>This document is an example of an HTML 4.01. In this instance, our goal is to create Web documents that are valid.</p>
</body>
</html>

DOCTYPE Declaration

The first line in our example is the document type declaration or <!DOCTYPE> declaration or <!DOCTYPE> indicator. This declaration line indicates the particular version of HTML being used, in our example HTML 4.01 Transitional. The <!DOCTYPE> declaration indicates to your browser the language being used in your document by referencing an external document type declaration (DTD). SGML describes other languages by specifying what is allowed in terms of structure through a DTD. Imagine the DTD as a blueprint controlling what kind of elements and attributes can exist in a document. Your browser uses the DTD to ensure the document's tags and organization agrees with the predefined conventions for HTML

Below are examples of various document type declaration:

XHTML-1.0-Strict

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

XHTML-1.0-Transitional

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">

XHTML-1.0-Frameset

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Frameset//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-frameset.dtd">

HTML-4.01-Strict

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

HTML-4.01-Transitional

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

HTML-4.01-Frameset

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">

According to the W3C the document type,"defines which version of (X)HTML your document is actually using, and this is a critical piece of information needed by browsers or other tools processing the document." The <DOCTYPE> contains the name of the Document Type Definition (DTD), public and system identifier. The public identifier denotes who published the DTD and the identity of the DTD such as XHTML 1.0 Transitional or HTML 4.01 Strict. In addition, the public identifier indicates which language it is written in, e.g., EN means English. The system identifier (http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd) is the address of the actual document type definition such as the W3C's Web site.

Although the <DOCTYPE> declares what flavor of HTML is being used it does not instruct how a browser (user agent) should interpret a web document. That is left to the "header" in the Hypertext Transfer Protocol (HTTP) specified by the "Content-Type." What is important about the <DOCTYPE> is document or doctype switching. Most modern browser use the <DOCTYPE> to determine how a Web document should be rendered. When a browser encounters a valid <DOCTYPE> it switches into a "standards mode" ensuring the document will be displayed according to the W3C recommendations. If you do not include a recent <DOCTYPE> or an old one the browser will switch into a "quirks mode" assuming invalid markup and code. Your goal therefore, should be the compliant or standards mode.

For a more complete list of document types navigate to:  List of valid DTDs

HTMLElement

Immediatly following the document type declaration is the HTML element. The beginning and end of a Web document is contained within the <html> tags. The html element is sometimes referred to as the root element.

HEAD Element

The head section is deimited by the <head> tags. This section contains information about the document such as its title and keywords used by search engines. The head may contain the following element types:

  • Script: Places script code in the document.
  • Style: Allows to insert style sheet rules.
  • Link: Defines a link
  • Object: Controls how data is to rendered such a through a program.
  • Base: Resolves relative URIs.
  • Meta: Identifies properties of the docment.

Tip: You can improve your search engine rankings by including a good title. A good title will use as many keywords as possible to descibe your document.

BODY Element

The content or data we see onscreen via our browsers is contained within the <body> tags. Between the <body> tags you can construct text into paragraphs, create lists, provide links to other Web sites, horizontal rules, and so on.

web page creation

Since a Web document is nothing more than a simple text file you can create a Web page using a standard text editor program such as Notepad. It involves adding the tags and content you want and then saving the document with either a "htm" or html" file extension. For example, copy the markup below and paste it into Notepad. Then, save the document as described above to your hard drive. To view the Web page simply open it using your default browser.

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">
<html lang="en">
<head>
<title>Creating a Web Page</title>
</head>
<body>
<h1>HTML Web page</h1>
<hr>
<p>The Declaration of Independence affirms moral obligations.
<br>
All men are created equal, endowed with certain unalienable Rights.
<p>The Bill of Rights was written to ensure the moral obligations of the Declaration of Independence are not abridged.
</body>
</html>

The above example illustrates a simple HTML 4.01 Web document. In keeping with this standard notice the two instance of the <p> tags do not have associated end tags. Moreover, the two empty tags, horizontal rule or <hr> and line break or <br> are enclosed in angle brackets.

To transition the above document to one that conforms to the XHTML standards we only are required to make a few minor changes.

<?xml version="1.0"?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en">
<head>
<title>Creating a Web Page</title>
</head>
<body>
<h1>HTML Web page</h1>
<hr />
<p>The Declaration of Independence affirms moral obligations.
<br />
All men are created equal, endowed with certain unalienable Rights.</p>
<p>The Bill of Rights was written to ensure the moral obligations of the Declaration of Independence are not abridged.</p>
</body>
</html>

In the above example notice that the <DOCTYPE> was changed to indicate a XHTML document and that all start tags have associated end tags. In addition, the two empty tags are closed by inserting a space and forward slash before the closing angle bracket. The first line is known as an XML declaration (specifying character encoding), which the XHTML standard strongly recommends. However, inserting the XML declaration may trigger some browsers into a non-standards compliant rendering or quirks rendering. To avoid this problem simply remove the XML declaration and specify the character encoding in the "head" section with a meta tag such as shown below.

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml" xml:lang="en" lang="en">
<head>
<title>Creating a Web Page</title>
<meta http-equiv="Content-Type" content="text/html; charset=us-ascii" />
</head>
<body>
<h1>HTML Web page</h1>
<hr />
<p>The Declaration of Independence affirms moral obligations.
<br />
All men are created equal, endowed with certain unalienable Rights.</p>
<p>The Bill of Rights was written to ensure the moral obligations of the Declaration of Independence are not abridged.</p>
</body>
</html>
