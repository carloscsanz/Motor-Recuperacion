Tags:XHTML

This page provides a list of tags in the XHTML 1.0 specification.

If you're looking for HTML 4.01, look Tags:HTML.

Note: XHTML elements MUST be lower-case (except for DOCTYPE which may be capitalized as per the examples below). Attribute values MUST be enclosed in either single or double quotes ("value" or 'value' but NEVER value). Example: <meta name=author content=me> is valid in HTML, but in XHTML it must be <meta name="author" content="me" /> or <meta name='author' content='me' />.

Start/Head Tags

These tags include:

  • <!DOCTYPE> -- Defines HTML version/edition
    • <html> -- Opens/closes document
      • <head> -- Head information
        • <title> -- Sets the title of the page
        • <meta> -- Additional page information
        • <link> -- Used for RSS feeds or external stylesheets
        • <style> -- Internal stylesheet
      • <body> -- Begins the actual page rendering

In XHTML 1.0 ("XHTML" from now on in this document), you would use DOCTYPE for telling the browser the version and edition of the XHTML specification. Example: 

<!DOCTYPE html PUBLIC "-//W3C/DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

Now you can use any one of Transitional, Strict, or Frameset. I just prefer to use Strict most of the time.

The html tag begins the document. Everything except DOCTYPE goes in between these tags. 

<!DOCTYPE html PUBLIC "-//W3C/DTD XHTML 1.0 strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html>
  <head>
    <title>Page Title</title>
    <meta http-equiv="Content-Type" content="text/html; charset=UTF-8" />
    <link rel="stylesheet" href="includes/style.css" type="text/css" />
    <style type="text/css">
    .test {
      background: #00000;
    }
    </style>
  </head>

  <body>

  </body>
</html>

Meta Tag

Used for providing extra information such as character set and content-type of the page. It can also be used to add a description and keywords for search engines. There are basically three attributes it can have:

  • http-equiv
  • name
  • content

The http-equiv and name attributes are mutually exclusive. If you use one, you don't use the other. The http-equiv attribute can be used for redirects, encoding information, or other HTTP headers.

You may also notice a few other slight differences from HTML. Particularly the /> as the closing. This is because XHTML requires closing every element and not every tag has a closing tag. It's what's called a short tag. The line break element is another example. The Link element is also another. Because of this, these tags must be closed with />.

<meta name="author" content="contact@thescripters.com" />

Link Tag

This tag is used for linking external stylesheets (.css) or RSS feeds to your page. The attributes are:

  • rel (relationship -- can be one of "stylesheet" or "alternate")
  • type (Content-Type -- for stylesheet "text/css" for RSS "application/rss+xml" or "application/atom+xml")
  • href (location of the file relative to the page)

The Link tag is a "short" tag and must be closed with />.

Style Tag

The style tag is used for defining internal stylesheets. The required attribute is type and should be set to "text/css" as per the example above.

Body Tag

This is what surrounds the actual "body" of the page. Anything visible should be within this tag. If there's something visible that is outside this tag, you have a problem (such as an open tag).

Block Level Tags

Block Level elements are those elements that don't require being inside another. For example, p can be on its own and does not need to be inside a div element. Likewise, block elements cannot be inside inline or formatting elements. Here are a few "block" elements:

  • div
  • p
  • h1
  • h2
  • h3
  • h4
  • h5
  • h6
  • table
  • form

Some of these are discussed later in this document. For now, let's focus on div and p.

Div Tag

The div tag is a special element, in that it serves as a "container" or a "box" for other elements. It will not go inside another element, with the exception of the form element. Every other tag (including the form element) will go inside of the div element. The div element has become the new "top-block" element recently. It used to be that an entire website would be laid out with tables and cells. But this could get confusing, and so now the div tag takes care of it.

The attributes that this tag requires are:

  • name
  • id
  • class

Generally, if you use id you don't necessarily need to use class, and name is basically just for compatibility with older browsers. In the Strict specification, the name attribute doesn't even exist, and has been deprecated in favor of the id. Example: 

<div id="main">text/markup</div>

In the CSS (Cascading Style Sheet), you would use .main to specify a class, but with an element's id, you would use #main instead.

P Tag

The p tag is used to define paragraphs. It allows you specify first line indentation without using extra spaces as well as split up text by placing a margin between blocks of text.

Formatting Tags

These tags include:

  • b -- bold
  • i -- italic
  • u -- underline
  • strike -- strikthrough
  • center (used only in HTML <= 4.01, invalid in XHTML)
  • strong (alternative to b) -- bold
  • span (preferred method for most formatting)

Span Tag

Tables

Tables have their own set of tags. In fact there are about four separate ones:

  • table
  • tr
  • th
  • td

Each has its own use. The table tag opens or closes a table: 

<table>

</table>

The tr tag opens or closes a row. Table Row: 

<table>
  <tr>

  </tr>
</table>

The th tag is for a Table Heading: 

<table>
  <tr>
    <th>Heading 1</th>
  </tr>
</table>

And finally, td is for each cell, or Table Data: 

<table>
  <tr>
    <th>Heading 1</th>
    <th>Heading 2</th>
  </tr>
  <tr>
    <td>Information 1</td>
    <td>Information 2</td>
  </tr>
</table>

This table would look like:

Heading 1 Heading 2
Information 1 Information 2

You may notice that the code given does not really give a border, as this one did. To do that, use the attribute border="1". There are others that can be given to each element:

  • border
  • rules
  • frame
  • width
  • height
