    Hi

    I have a TreeView placed in a LoginView on my MasterPage. The TreeView is connected to a Web.sitemap file which generates the Text displayed by the TreeView MenuItems. I would like to be able to programmatically change the Text displayed by the TreeView MenuItems. Can that easily be done? My TreeView looks like this:

    <asp:TreeView ID="masterTreeView" runat="server" DataSourceID="SiteMapDataSource1"
                            NodeIndent="0">
                        </asp:TreeView>
                        <asp:SiteMapDataSource ID="SiteMapDataSource1" runat="server" />


     

    My Web.sitemap looks like this:

    <?xml version="1.0" encoding="utf-8" ?>
<siteMap xmlns="http://schemas.microsoft.com/AspNet/SiteMap-File-1.0" >
  <siteMapNode url="Member.aspx?home=" title="Home"  description="Start Page">
    <siteMapNode url="Search.aspx" title="Search"  description="Search Page" />
    <siteMapNode url="MessageIn.aspx" title="Messages Inbox"  description="Messages Inbox" />
    <siteMapNode url="MessageOut.aspx" title="Messages Outbox"  description="Messages Outbox" />
    <siteMapNode url="FriendRequest.aspx" title="Friend Requests"  description="Friend Requests" />
    <siteMapNode url="Friend.aspx" title="Friends"  description="Friends" />
    <siteMapNode url="AccountInfo.aspx" title="Edit Profile"  description="Account Info" />
    <siteMapNode url="ShowMessage.aspx?admin=" title="Contact Admin"  description="Contact Admin" />
  </siteMapNode>
</siteMap>

    The thing I would like to do is quite common, the siteMapNode "Messages Inbox" is the message inbox of my system, in case of that there is unread messages I would like that to show in the TreeView. I don't know how to do it. Is it perhaps the siteMapNode of the Web.sitemap which should be accessed instead of the TreeViewNodes???

    If you have PopulateOnDemand=true, You can write code for the TreeNodePopulate event and append the number of unread messages to the tree node text:


        protected void TreeView1_TreeNodePopulate(object sender, TreeNodeEventArgs e)
    {
        if (e.Node.Text == "In-Box")
        {
            e.Node.Text = string.Format("In-Box ({0})", unreadMessageCount);
        }
    }

    Otherwise, you will have to handle the Databound event and update the node in there.

    Hi,

    You have to React to Treeview's  TreeNodeDataBound Event.

    <asp:TreeView OnTreeNodeDataBound="Data_Bound" />

    This will be called when a data item is bound to a node in the TreeView control.

    Then in that Event,you can check your condition,and change the NodeText.

    void Data_Bound(Object sender, TreeNodeEventArgs e)
  {

 
    if(e.Node.Text == "Chapter 2")
    {
      e.Node.Text="Custom";
    }

  }

    Also refer :-

    http://msdn.microsoft.com/en-us/library/system.web.ui.webcontrols.treeview.treenodedatabound.aspx

    I hope this helps...

    If you are getting a NullReferenceException, that is probably because the masterLoginView.FindControl method cannot find the treeview you are looking for.

    Check to see if masterTreeView is null:

    protected void Page_Load(object sender, EventArgs e)  
{  
        TreeView masterTreeView = (TreeView)masterLoginView.FindControl("masterTreeView");  
        if (masterTreeView != null)
            masterTreeView.PopulateNodesFromClient = true;  
}

    This will determine if the problem is with your masterLoginView or not.
