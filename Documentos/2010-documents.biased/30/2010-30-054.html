Hide any closing tags like '</td>' that are within document.write statements by either breaking them open '</' + 'td>' or by escaping them <\/td> and the validator will no longer complain about these closing tags. It is important to break directly after the / and not the middle of a word.

For the XHTML tag <br /> you will need <br \/> instead.

It is not valid to put <!-- comment --> tags around the javascript code in XHTML. You may find that some browsers completely ignore your javascript code. Remove the comment tags.



I must say that I see no reason why this has to be done as XHTML when HTML 4 would have sufficed. XHTML has problems depending on which MIME type you serve it with. It should be served as XML but doing that breaks most current browsers.

Only in XHTML 1.0 strict or XHTML 1.1 should you set the mime-type to XML. Setting it to this causes problems with a lot of browsers anyway. Therefore set it to text/html (which validates for XHTML 1.0 Transitional). If you want to still validate as XHTML1.1 then change copy the DTD and change it to accept text/html.

Also, it is highly recommended when embedding javascript code to create a source file and call it using the src attribute (as opposed to embedding it inline). This is because strictly speaking javascript text is CDATA (because you would otherwise need to use a lot of entities, etc).

And Scoutt re: your comment ("now I know why I hate XML") this has nothing to do with XML, the rules you refer to are rules from the SGML of the HTML 4.01 specification and not XHTML (and certainly not any XML (which has no knowledge of predefined HTML tags)).

Dissecting this further, take a look at the HTML (not XHTML) specification. We see this definition of the <body></body> element<!ELEMENT BODY O O (%block;|SCRIPT)+ +(INS|DEL) -- document body -->
<!ATTLIST BODY
%attrs; -- %coreattrs, %i18n, %events --
onload %Script; #IMPLIED -- the document has been loaded --
onunload %Script; #IMPLIED -- the document has been removed --

The first line is the one we are interested in for our example. Notably the part which says:

(%block;|SCRIPT)+

This means that the opening <body> tag MUST be followed by either a <script> element or, more importantly in our case, an element defined in the %block entity. The + means that one or more of these must occur, but whatever the case there must at least be one of them!

Now, let's take a look at the %block entity.<!ENTITY % block
"P | %heading; | %list; | %preformatted; | DL | DIV | NOSCRIPT |
BLOCKQUOTE | FORM | HR | TABLE | FIELDSET | ADDRESS">
Only the elements which are stated here are permissable after the <body> tag. Some are explicitly stated, such as the <p> element, and some are implied through further entity references, such as %heading. One thing you will note (if you follow the entity references) is that it is not HTML 4.01 compliant to follow a <body> tag with a <br> element.

So, in short, the failure both you and I observe is a failure to comply with the HTML4.01 specification, and not the XHTML one. In my case this is because the <body> element is empty (not permissable), and in your earlier post's example, because the <body> element contained a <br> element, also not permissable.
