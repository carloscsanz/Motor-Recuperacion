Dynamic tags in mail templates

IBM® Tivoli® Identity Manager mail templates allow dynamic retrieval, substitution, and decision making in creating a message.

Dynamic content tags and examples

Tivoli Identity Manager provides dynamic content tags to allow text substitution and enable translation. The tags are used for the e-mails generated by these tasks:

  • Designing workflows
  • Specifying mail activity
  • Manual service notification
  • Recertification notification
  • Post office
  • Reminder template
  • Default system notifications

The tags include:

JavaScript™ code
Handles JavaScript and executes the JavaScript content contained between the open and close tags. This tag contains child tags unless they return a string. JavaScript code is called within <JS>MyJavaScriptCode </JS> delimiters.
Table 1. Syntax and example of using JavaScript code to replace message content
Syntax Example 
<JS>text or JavaScript tag</JS>

Enter each <JS></JS> statement as one line:

An account request has been initiated for
<JS>process.requesteeName;</JS>
<JS>if (var x=process.getParent() !=null)
 return x
</JS>
 
<JS escapeentities="false">text or JavaScript tag</JS>

When specified as "false", any text returned by the JavaScript execution will not have its HTML entity tags escaped. For instance, the < character will not return as &lt;. This option might be useful when the execution of the JavaScript code would return XML (such as embedding XHTML body notifications inside the XHTML body of the post office template).

The default for this attribute is "true", so not specifying the tag will result in the normal behavior.
Replace tag
Formats the message represented by the key, allowing string replacement. The formatted string can have zero or more parameters. Parameters can contain strings, activity IDs, or JavaScript. The string inside the key must exist in the CustomLabels.properties file. Strings are sourced from a CustomLabels.properties resource bundle file or from the Labels.properties file.

The key of the string replacement can be specified using the key attribute or by adding a KEY tag between RE tags. Specifying a key that uses both the attribute and tag at the same time will result in an exception.

The tag has these parameters:

Key tag
Represents the resource bundle key for a RE tag. For example:
<RE key="key">
</RE>
PARM tag
Represents the parameters for a RE tag. For example:
<RE key="key">
<PARM>with plain text</PARM> 
</RE>
Table 2. Syntax and examples of using a RE tag to replace message content
Syntax Example 
<RE key="message key">
<PARM>text or JavaScript tag</PARM> 
</RE>

or (enter each <KEY></KEY> statement as one line):

<RE><KEY>message key or 
 JavaScript tag to return a key
</KEY>
<PARM>text or JavaScript tag</PARM> 
</RE>

The KEY can be specified by either an attribute on the RE tag, or as a sub-element of the RE tag using the tag KEY.

 
<RE key="message key">
<PARM>with plain text</PARM> 
<PARM><JS>process.requesteeName;
</JS></PARM></RE>

Output:

This is a formatted string replacement
example with plain text and
JavaScript code for requestee name
John Smith.
To enable string replacement for translation, specify a custom label in a CustomLabels.properties file to overwrite a Labels.properties key.
For example, the Labels.properties file contains this key/value pair: 
readOnlyDateFormat=MMM dd, yyy hh:mm:ss z

To override this format, add the same key to the CustomLabels.properties file.

 
<RE key="readOnlyDateFormat">
<PARM><JS>if (process.scheduled !=null) 
 return process.scheduled.getTime();
   else
     return "";</JS></PARM></RE>

Output:

Apr 18, 2005 05:20:52 EDT

ITIMURL tag
Based on person's group membership, represents the URL of the Tivoli Identity Manager. A forced URL can be applied using "forcedurl" attribute of the tag. This attribute contains constant values such as the value console or the value enduser.
Table 4. Syntax and examples of ITIMURL
Syntax Example 
<ITIMURL/>

Based on person's group membership, represents the URL of the Tivoli Identity Manager Server.

 
<ITIMURL forcedurl="enduser"/>

Represents the URL of the graphical user interface on the Tivoli Identity Manager Server. If the forcedurl attribute is used, the URL is not generated on the basis of person's group membership.

Values for this attribute include:

enduser
The URL points at the self-service graphical user interface.
console
The URL points at the administrator graphical user interface.

Properties file values

To change templates, you can add the key=value statements in the CustomLabels.properties file, or create your own properties and values.

Required escape characters and JavaScript

The following characters need to be escaped (using the appropriate HTML entity form that has the format &entity;) to ensure that the notification template XML is well formed.

Table 5. Escape characters
Escape character Character
&lt; Less Than (<)
&gt; Greater Than (>)
&quote; Quote (")
&apos; Apostrophe (')
&amp; Ampersand (&)

For example, to use the following JavaScript:

if (i<4) return "less than four";

The dynamic content tag would be formed as:

<JS> if (i&lt;4) return &quote;less than four&quote;;</JS>

Common formatting patterns in the XHTML body

Default messages are formatted with a common pattern in the XHTML body, and also contain message-unique statements.

For example, the XHTML for the to-do reminder template calls a common stylesheet (the imperatives.css file) and logos. Message-unique statements are similar to these:

<!-- Start of notification body -->
						<textBody/>
  						<RE key="escalation_note"/> <escalationTime/>
					</td>	
				</tr>
<!-- End of notification body -->
